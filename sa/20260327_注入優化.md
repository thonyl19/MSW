# 2026-03-27 MSW 注入功能與面板架構優化方案

對於注入功能及 MockPanel 的優化規劃，經過討論後達成以下共識：

## 1. 資料偵測與註冊機制 (Auto-detection)
*   **一致性命名：** 採用 `[Page].js` ↔ `[Page].Data.js` 的 1:1 對應慣例。
*   **自動載入：** 當頁面註冊時，系統依據上述慣例自動嘗試載入對應的 `.Data.js` 檔案。若檔案內含 `_inject` 集合則自動生成注入 UI，否則略過。
*   **多來源管理：** 支援載入多個 Data 檔案，並在介面上提供動態切換。

## 2. MockPanel UI 佈局優化
*   **頂部固定控制區：** 將「熱重載 (Reload)」、「MSW 開關」、「縮小按鈕」以及新加入的「Catch/Clear Catch」統一置於面板上方，與下方 Tabs 分離。
*   **面板規格：** 面板適度加寬（建議 420px），以提供更佳的閱讀與操作空間。
*   **動態來源切換器 (Source Switcher)：** 在 Tabs 上方或同一列增加切換按鈕/選單：
    *   **A. 單一來源：** 不顯示切換器。
    *   **B. 兩個來源：** 以 Radio Buttons 模式呈現。
    *   **C. 超過兩個來源：** 以 Select 下拉選單呈現。
*   **分頁架構 (Tabs)：**
    1.  **Tab 1 - "MSW"：** 處理 MSW 案例情境及基礎 API 控制（Delay, Status）。
    2.  **Tab 2 - "Inject"：** 處理資料注入功能。注入按鈕（Action Buttons）樣式微縮，並支援按 `target` 分組與折疊 (Collapsible)。
*   **搜尋過濾：** 在 "Inject" 分頁加入關鍵字過濾功能，並連動折疊區塊自動展開。

## 3. 持久化機制 (Catch System)
*   **手動 Catch：** 採手動觸發模式，點擊 "Catch" 鈕後將目前設定儲存。
*   **URL 隔離：** 使用當前 URL（不含 Domain 與 Query Parameters）作為辨識碼：`msw_catch_${window.location.pathname}`。
*   **Catch 範圍：** 包含 MSW 開關、API 設定以及各來源的動態選項值。
*   **持久化介質：** 預計採用 `sessionStorage` 以對應開發測試週期。

## 4. 技術規格與限制
*   **注入屬性：** 目前不考慮巢狀屬性，一律以根節點為主。
*   **資料完整性：** `_inject` 內的 `value` 不應且不會有 `null` 情形。
*   **風格樣式：** 統一使用現有的深色溢彩風格 (Rich Dark / Glassmorphism)，並改以動態 `<style>` 標籤代入 CSS。
