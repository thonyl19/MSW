## AI 小助手 初步功能企劃

### 1. 產品定位與目標
- **定位**：基於 Web (Vue 2) 的嵌入式浮動智慧助理系統。旨在提升使用者在系統中的優質操作體驗。
- **目標**：透過看板娘形象增加互動親和力與趣味性，提供即時的頁面輔助與問題解決方案，降低系統操作的學習門檻及客戶服務負擔。

### 2. 介面設計與互動形式 (UI/UX)
- **浮動對話框 (ChatGPT 樣式)**：
  - 常駐於頁面角落（可固定於右下角或允許拖曳），支援點擊展開與最小化收合。
  - 介面風格考量資源與開發成本，初期以類似 ChatGPT 的簡潔對話 UI 為主，不強求 Live2D 等複雜動畫。
  - 收合時顯示專屬圖示（Icon）或提示泡泡（Bubble），提示未讀訊息或操作建議。
- **(選配/暫緩) 看板娘模式 (Virtual Assistant/Live2D)**：
  - 若未來資源允許，可引入基礎視覺互動動畫（如：待機、點擊互動）作為系統引導精靈。

### 3. 核心功能模組 (Features)

#### 3.1 頁面操作指引 (Contextual Help / Walkthrough)
- **指定載入 YAML 配置機制 (類 MSW 模式)**：由前端依據當前畫面需求，主動指定並載入相對應的 YAML 導覽腳本（而非僅依賴 Route Path 自動配對）。
- **YAML 視覺化導覽 (基於 Driver.js)**：透過外部抽離的 `.yaml` 設定檔（定義要反白的 element 選擇器、標題、內文），將參數交由 `Driver.js` 負責繪製畫面遮罩與引導泡泡，達成「不影響現有 HTML 版面」即可實現的頁面教學。
- **名詞解釋與提示**：在特定欄位旁提供快速問答，或主動推播實用功能的小撇步。

#### 3.2 支援問題回報 (Issue Reporting)
- **一鍵擷取與回報**：使用者出現異常問題時，可透過指定 Icon 或小助手介面觸發報修機制，自動擷取當前頁面異常資訊（截圖、前端日誌等）。
- **發送 Email 通知**：將夾帶上下文的異常資訊，以單純發送 Email 的方式通知對應窗口。（個資隱私安全與截圖遮罩等進階功能，目前做為立項保留項目，暫不實作。）

#### 3.3 聯絡窗口指引 (Contact Support)
- **支援管道串接**：若簡單除錯無法解決問題，主動提供明確的下一步聯絡資訊（專責窗口 Email 等）。
- **處理排解進度查詢**：讓使用者透過小助手，快速查詢以往回報問題的處理進度。

#### 3.4 基礎問題排除與查詢 (Troubleshooting)
- **關鍵字 FAQ**：目前暫不計畫接入複雜的 AI (LLM) 對話模型，初期以「簡單關鍵字搜尋」或標籤點選為主，給予對應排解步驟。
- **異常主動通知入口**：透過前面提到的報修機制，讓面臨操作錯誤的使用者能快速將資訊外送。

### 4. 技術與架構考量 (實作備註)
- **介面實作**：以 Vue 2 實作輕量級的對話框 (類似 ChatGPT UI)，不導入會拖慢效能的 Live2D 以節省專案資源。
- **頁面感知與報修**：依賴 `window.location` 取得當前網址狀態，並做為異常發信通知的上下文依據之一。
- **導覽機制核心套件**：引入 `driver.js` 作為畫面高亮導覽引擎，引入 `js-yaml` 進行靜態劇本解析，以利 PM 或開發編譯維護脫鉤。

### 5. 建議的推動階段 (Phases)
- **Phase 1 (基礎介面與回報 POC)**：完成基於 Vue 2 的浮動對話面板 (ChatGPT style)，實作「擷取頁面異常狀態並寄發 Email」功能，以及依據 `window.location` 動態呈現當前頁面輔助文字。
- **Phase 2 (動態導覽與 FAQ)**：
  - 優先實作：基於 `Driver.js` + `js-yaml` 的操作指引框架，採用類 MSW 之「前端指定載入 YAML 檔」邏輯建立第一個導覽 POC。
  - 建立系統知識點與問題集，透過簡單關鍵字比對機制，達成基礎的自助查詢功能。
- **Phase 3 (後續保留項目)**：看板娘動態渲染導入、自動隱私/個資遮罩機制、以及未來可能介接 LLM 模型的真實 AI 對話能力。
