# 技術分析：MSW 熱重載 (Hot Reload) 方案

## 1. 目標 (Objective)
為 `MockPanel.js` 增加熱重載功能，允許在不重新整理網頁的情況下，直接獲取更新後的 `pages/*.js` 與 `.data.js` 數據，提升開發效率。

## 2. 核心技術挑戰
*   **ESM 瀏覽器快取**: 瀏覽器會快取已載入的模組，重複 `import` 同一 URL 不會重新抓取。
*   **依賴鍊 (Dependency Chain)**: 若 `page.js` 引用了 `data.js`，僅重載 `page.js` 無法確保 `data.js` 同步更新。
*   **MSW Handler 堆疊管理**: 需要在重載時清理舊的攔截器，避免邏輯衝突與記憶體洩漏。

## 3. 最終方案選定

### A. 觸發機制
*   **手動觸發**: 在 `MockPanel.js` 增加一個「🔄 重載 Mock (Hot Reload)」按鈕。

### B. 模組更新機制 (深度解析)
*   **源碼重寫 + Blob URL 注入**:
    1.  使用 `fetch()` 抓取頁面 Entry Point 的原始碼。
    2.  使用 Regex 遍歷所有相對引用 (`./...js`)，強制加上 `?t=TIMESTAMP` 時間戳。
    3.  將重寫後的內容轉換成 `Blob URL` 並執行 `import()`。
    4.  這將強制瀏覽器重新抓取整條依賴路徑上的所有檔案。

### C. MSW Handler 管理
1.  重載前呼入 `worker.resetHandlers()`。
2.  重新載入 `src/handlers.js` (全域邏輯)。
3.  重新載入受追蹤的頁面模組，觸發其內部的 `registerMock()`。

## 4. 討論事項記錄 (2026-03-27)

| 項目 | 決策/結論 |
| :--- | :--- |
| **自動/手動** | 採用手動按鈕觸發。 |
| **依賴處理** | 採用「複雜解析機制 (Regex Rewrite + Blob URL)」，確保 `.data.js` 能同步更新。 |
| **路徑追蹤** | 由 `msw-loader.js` 負責將載入的頁面路徑註冊至 `store.js` 的 `loadedPages` 中。 |
| **全域邏輯** | 重疊時必須確保 `src/handlers.js` 被重新註冊，維持登入等基礎功能。 |
